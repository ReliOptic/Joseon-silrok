## 1. 한 문장 요약

> 500년 27왕의 역사를 한 화면에서 줌인/줌아웃으로 파고드는, 한 번 들어오면 30분 빠져나오지 못하는 **인터랙티브 사관**.

학습 콘텐츠가 아니다. 콘텐츠 경험이다.
교과서가 아니라 한 편의 영화처럼 끝까지 보게 만드는 것이 목표다.

---

## 2. 본 의뢰의 위치와 기대 수준

이 앱은 이미 기능적 골격이 완성되어 있다. 27 왕의 데이터, 5단계 줌 UX, URL 동기화, 검색, 공유까지 모두 작동한다.

이 의뢰는 **"한 번 더 만들어달라"가 아니라 "이 골격을 완성된 작품으로 끌어올려달라"** 이다.

기대 수준의 기준점:
- Apple Newsroom의 깊이감
- The Pudding의 인터랙티브 에디토리얼
- 일본 NHK 다큐 그래픽의 절제
- 국립중앙박물관 디지털 아카이브의 권위감

이 네 가지가 한 자리에서 만나는 지점이 우리가 원하는 자리다.

본 의뢰는 단가, 일정, 디자이너 본인의 자유로운 해석 영역에 대해 협의 가능하다. 그러나 **퀄리티 기준은 협의 대상이 아니다.** 의뢰 결과물은 디자이너의 포트폴리오에 대표작으로 올릴 수 있는 수준이어야 한다.

## 3. 타겟 사용자

### 1차 페르소나 — "역사가 좋은 30대 직장인"
- 점심시간이나 출퇴근 지하철에서 모바일로 5분 들어왔다가 30분 머문다
- 사극을 보고 위키피디아로 인물을 찾아본 경험이 있다
- 짧은 카피 한 줄에 강하게 반응한다
- "공유하고 싶다"가 핵심 행동

### 2차 페르소나 — "조선사 깊이 파고드는 학습자"
- 데스크탑에서 진지하게 본다
- L4 한문 원문까지 본다
- 출처/근거 표기에 민감하다

### 3차 페르소나 — "외국인 한국문화 관심층" (Phase X)
- 영문 인터페이스 (현재 미지원, 디자인 시 다국어 확장 가능성 고려)

## 4. 현재 상태 (As-Is)

### 4.1 5단계 줌 UX

```
L1 (Macro)    시대 카드 + 27왕 그리드
   ↓ 클릭/줌인
L2 (Events)   특정 왕의 사건 타임라인
   ↓
L3 (Detail)   사건 상세 + 야사 + 인물
   ↓ (storyEntry 있을 때만)
L3.5 (Story)  AI 내레이션 씬 (다크 그라디언트, 단일 씬 표시)
   ↓
L4 (Sillok)   실록 원문 + 번역 + 사신 평
```

**진입 후 행동**: GSAP 줌 애니메이션 + URL 동기화. 핀치/Ctrl+휠 제스처도 가능.

### 4.2 현재 비주얼 언어 (이미 구축된 디자인 시스템 일부)

| 요소 | 현재 구현 |
|---|---|
| 배경 | `hanji-noise` 노이즈 텍스처 + 시대별 색상 변수 |
| 시대 색상 | 1기 청록(`#A5CCBF`), 2기 회백(`#E2E7E4`), 3기 청남(`#1F3A69` 보색), 4기 미색·금황(`#F5EFE6`/`#F7B500`) |
| 카드 | `bg-white/40 backdrop-blur-md` 글래스 + `rounded-3xl` |
| 한자 워터마크 | 카드 우상단 `text-[200px] opacity-5` |
| 야사 박스 | 미색 종이(`#F5EFE6`) + 노란 보더 + 좌상 꼬리 (말풍선) |
| L3.5 다크 모드 | `from-[#0d0d1a] via-[#111827] to-[#0a0a12]` 가로 16:9 |
| Typography | 영어 sans-serif 기본 + `font-serif`(역사적 임팩트) + `font-mono`(기술 메타) |

이 언어는 유지하되 더 정교하게 다듬어주길 원한다. 통째로 갈아엎지 말 것.

### 4.3 알려진 약점 (디자이너가 풀어야 할 문제)

1. **왕 ↔ 왕 전환이 단순 swap** — 0.4초 fade밖에 없다. 27 왕이 따로 노는 느낌.
2. **L1 정보 위계 불분명** — 시대 카드와 왕 그리드가 동등 비중으로 보인다.
3. **TimelineBar의 존재감이 약함** — 항상 떠있는데 어떤 도움이 되는지 모호.
4. **L3 → L3.5 → L4 사이 시각적 단절** — 라이트(한지) → 다크(우주) → 라이트(연두)로 너무 급격하다.
5. **모바일 첫인상이 약함** — 320 px 환경에서 시대 카드가 무너진다.
6. **공유 카드(OG)가 없음** — SNS에 붙이면 그냥 텍스트 링크다.
7. **로딩/전환 상태 디자인 없음** — Phase 3 코드 스플리팅이 들어가면 반드시 필요.

### 4.4 주요 컴포넌트 코드 위치 (디자이너 참고용)

```
src/App.tsx                              # 줌 상태 머신, 네비게이션
src/components/Level1MacroView.tsx       # 시대 + 왕 그리드
src/components/Level2EventView.tsx       # 사건 타임라인
src/components/Level3DetailView.tsx      # 사건 상세
src/components/Level35StoryView.tsx      # AI 스토리 씬
src/components/Level4SillokView.tsx      # 실록 원문
src/components/TimelineBar.tsx           # 하단 타임라인
src/components/SearchModal.tsx           # 검색
src/index.css                            # CSS 변수 + hanji-noise
```

라이브로 보려면: `npm install && npm run dev` (port 3000)

## 5. 의뢰 범위 (Scope)

다음 7개 영역을 디자인한다. 각 영역은 독립 의뢰도 가능, 통합 의뢰도 가능.

### 5.1 [필수] 디자인 시스템 정립

현재 비주얼 언어를 코드/Figma에 정식 토큰으로 정리한다.

- 색상 토큰 (시대별, 레벨별, 의미별 — 한지/먹/단청/금박 계열 정확화)
- 타이포 토큰 (한글/한자/영문 각각의 위계 — 본문/제목/카피/메타)
- 간격, 라운딩, 그림자, 노이즈 텍스처
- 다크/라이트 페어링 (L3.5 다크 모드의 정확한 이론적 근거)
- 한자 워터마크의 사용 규칙

산출물: Figma 라이브러리 + 토큰 export (CSS 변수 호환)

### 5.2 [필수] 5 레벨 시각 정합성 재정립

L1 → L2 → L3 → L3.5 → L4가 **하나의 작품**으로 보이게.

- 레벨 간 색온도 전환의 의미적 근거 (예: 시대 → 인물 → 사건 → 환영 → 기록)
- L3 → L3.5 → L4의 시각적 다리 놓기 (현재의 단절 해결)
- 줌 애니메이션의 "물리적 실감" 강화 (현재는 단순 scale)
- 각 레벨의 grid, hierarchy, breathing room 재검토

### 5.3 [필수] 왕 ↔ 왕 전환 경험 디자인

이 의뢰의 **컨셉적 핵심**.

- 27왕 사이의 연결 고리 시각화 (혈통, 반정, 단명, 왕위 찬탈 등)
- L1에서 왕 클릭 시 진입 시퀀스 (현재 단순 swap)
- L2 끝에서 다음 왕으로 넘어가는 "이야기 다음 장" 모먼트
- TimelineBar의 역할 재정의 (현재 위치 표시 + 시대 색상 + 클릭 가능)
- 왕위 교체 방식별 시각 차별화 (정상 승계 / 반정 / 폐위 / 단명)

### 5.4 [필수] 모바일 디자인 (320 px ~ 768 px)

PC 우선으로 만든 앱이다. 모바일에서 살아남게 만든다.

- L1 시대 카드 모바일 레이아웃 재설계 (현재 그리드 무너짐)
- L3.5 씬 카드 16:9 → 4:5 또는 9:16 모바일 최적화
- L4 한문 원문 가독성
- 핀치 줌 vs 페이지 줌 충돌 해결을 위한 인터랙션 가이드
- 한 손 엄지 도달 영역 고려한 navigation

### 5.5 [필수] 로딩 / 전환 상태

Phase 3에서 코드 스플리팅이 도입되면 왕별 chunk 로딩 시간이 발생한다.

- 왕 진입 시 chunk fetch 동안 보여줄 "시간 흐르는 중" 모먼트
- 로딩 → 콘텐츠 등장의 매끄러운 전환
- 에러 상태 (네트워크 실패 등)
- skeleton vs ink-wash placeholder 중 우리 톤에 맞는 선택

### 5.6 [필수] OG 공유 카드 시스템

SNS 공유가 1차 페르소나의 핵심 행동.

- 사건별 OG 카드 템플릿 (예: "사도세자 — 뒤주의 8일")
- 왕별 OG 카드 템플릿
- 시대별 OG 카드 템플릿
- Vercel OG 또는 Cloudflare Workers SVG 베이스로 동적 렌더 가능한 형태

### 5.7 [선택, 장기] L3.5 이미지 스타일 가이드

> 이미지 생성은 본 의뢰 직후 진행 안 함. **스타일 가이드만** 산출.

- 수묵화 / 기록화 / 진경산수 중 우리 톤
- 컬러 팔레트, 인물 비례, 구도 원칙
- 부적절한 표현 (얼굴 디테일, 폭력성 등)
- 향후 Replicate/fal.ai 프롬프트 템플릿화 가능한 수준의 명세

## 8. 기술 제약 (지켜야 할 것)

- 프레임워크: React 19 + Tailwind CSS v4 (변경 불가)
- 애니메이션: GSAP 사용 중 (대안 제안 가능, 단 현재는 GSAP)
- 폰트 로딩: 한글 웹폰트 사용 시 용량 고려 (FOUT/FOIT 회피 필요)
- 다크모드: 시스템 자동 감지 + 강제 토글 모두 고려
- 접근성: WCAG AA 색 대비 (특히 L4 한문 본문)
- 반응형: 320 px ~ 1920 px 지원

## 9. 진행 방식 제안

### 1단계 — Discovery (1주)
- 라이브 앱 직접 사용 (`npm run dev`)
- 코드베이스 훑어보기 (`src/components/Level*View.tsx`)
- 현재 데이터 어디까지 들어갔는지 확인 (`src/data/sejong.ts`가 가장 풍부)
- 본 문서 + `ROADMAP.md` 정독
- 디자이너 본인의 1차 해석을 5장 이내 무드보드로 회신

### 2단계 — Direction (1주)
- 무드보드 합의 후 디자인 시스템 토큰 1차 정립
- L1과 L3.5 두 키 화면을 PC/모바일 각각 디자인
- 검토 후 톤 확정

### 3단계 — Build (3-4주)
- 5 레벨 전체 디자인
- 인터랙션 명세
- OG 카드 시스템

### 4단계 — Handoff (1주)
- Figma 라이브러리 정리
- 토큰 export
- 개발자와 1:1 핸드오프 미팅 (인터랙션 디테일 확인)

총 6-7주 기준. 단축 또는 확장 협의 가능.

함께 만들어주시길 부탁드린다.

— 의뢰자 드림
